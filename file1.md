# History about Continuous Integration(CI)

 - first era of computing ->
   - they was doing holes on a punch card then feed it into a computer
   - the developers who were building the software they were the consumers of these applications
  
 - Second era
   - it was about shipping the programs on floppy disks,CDs,DVDs ..
   - but this was a problem in itself as it took more time to ship the whole updates
   - also it costs alot of money.
  
   -third,fourth eras;
       - this was the time of high performance applications that are deployed many times per day
       - these deploying has automated checks before it reaches users.
now Github Actions -> is the CI platform that makes you start automation within a minutes .


# why Github Actions?
 - Low friction -> creating a workflow is as simple as creating a Yaml file
 - Virbrant marketplace -> there are much communities around it and they build pipelines that will enables you to automate your workflow faster.
 - also if your code live on Github so github actions will be suitable for your work.
